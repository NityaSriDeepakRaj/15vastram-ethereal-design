# 15Vastram — Luxury Indian Ethnic Wear

15Vastram is a premium e-commerce platform dedicated to handcrafted Indian ethnic wear. Bridging the gap between timeless heritage and modern aesthetics, it provides a seamless shopping experience for sarees, kurtas, lehengas, and more.

## ✨ Features

- **Modern Heritage Design**: A premium, minimalist aesthetic inspired by traditional Indian craftsmanship.
- **Dynamic Product Showcase**: A responsive grid to explore collections with fine detail.
- **Integrated Shopping Experience**: Seamless cart functionality for a smooth journey from discovery to purchase.
- **Handcrafted Collections**: Specialized sections for curated heritage pieces.
- **Responsive & Accessible**: Optimized for all devices, from mobile to desktop.

## 🛠️ Tech Stack

- **Framework**: [React 18](https://reactjs.org/)
- **Build Tool**: [Vite](https://vitejs.dev/)
- **Styling**: [Tailwind CSS](https://tailwindcss.com/)
- **UI Components**: [Shadcn UI](https://ui.shadcn.com/) & [Radix UI](https://www.radix-ui.com/)
- **Icons**: [Lucide React](https://lucide.dev/)
- **State Management**: [TanStack Query](https://tanstack.com/query/latest)
- **Routing**: [React Router DOM](https://reactrouter.com/)

## 🚀 Getting Started

### Prerequisites

- Node.js (v18 or higher)
- npm or bun

### Installation

1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd 15vastram-ethereal-design
   ```

2. Install dependencies:
   ```bash
   npm install
   # or
   bun install
   ```

### Development

Run the development server:
```bash
npm run dev
# or
bun dev
```

The application will be available at `http://localhost:5173`.

### Build

Build for production:
```bash
npm run build
```

### Testing

Run the unit tests:
```bash
npm run test
# or
bun test
```

### Linting

Run the linter to check for code quality:
```bash
npm run lint
```

## 📁 Project Structure

- `src/components`: Reusable UI components (Hero, Footer, Navbar, etc.)
- `src/pages`: Main application pages (Index, Shop, Lookbook, About)
- `src/contexts`: Application-level state (CartContext)
- `src/data`: Mock data for products and collections
- `src/assets`: Static assets like images and fonts
- `public/`: Public static assets
